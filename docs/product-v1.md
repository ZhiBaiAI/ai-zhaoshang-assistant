# V1 产品需求

## 产品定位

V1 是一套抖音招商私信自动接待系统，先解决客户咨询回复不及时、有效线索沉淀不完整、销售跟进不及时的问题。

第一版不做推广获客，不做完整平台，重点跑通一个闭环：

```text
抖音用户咨询 -> 自动回复 -> 引导留资 -> 线索入库 -> 飞书同步 -> 销售通知 -> 人工跟进
```

## 目标用户

- 招商加盟类商家
- 有抖音私信咨询流量，但缺少及时接待和线索沉淀能力的团队
- 销售团队已经使用飞书、企业微信、钉钉中的一种或多种工具

## V1 功能范围

### 1. 抖音私信自动化

- 读取抖音私信会话
- 识别新消息
- 消息去重
- 调用业务服务生成回复
- 自动发送回复
- 记录发送结果
- 登录失效和页面异常告警
- 支持人工接管开关

### 2. 本地知识库

- 支持项目资料录入
- 支持 FAQ 管理
- 支持禁用话术
- 支持留资引导规则
- 本地 embedding
- PostgreSQL + pgvector 向量检索

### 3. AI 回复引擎

- 用户意图识别
- 知识库召回
- 在线 LLM 生成回复
- 敏感内容校验
- 不确定问题转人工
- 多轮上下文摘要
- 留资引导

### 4. 留资识别

- 手机号识别
- 微信号识别
- 城市识别
- 意向区域识别
- 预算识别
- 开店时间识别
- 咨询重点摘要
- 意向等级判断

### 5. 线索中心

- 会话记录
- 消息记录
- 线索记录
- 意向等级
- 跟进状态
- 接待安排字段
- 销售负责人字段

### 6. 飞书多维表格同步

- 新线索创建飞书记录
- 线索更新同步飞书
- 飞书跟进状态定时同步回本地
- 同步失败重试

### 7. 销售通知

- 企业微信机器人
- 钉钉机器人
- 飞书机器人
- 新线索通知
- 高意向通知
- 留手机号/微信通知
- 请求人工通知
- 系统异常通知

## V1 暂不做

- 不做咨询页
- 不做复杂后台
- 不做完整 CRM
- 不做 OpenClaw
- 不做多平台
- 不做主动批量私信
- 不做复杂权限系统
- 不做数据大屏
- 不做内容生产和分发

## 验收标准

- 能稳定读取抖音新私信
- 能基于项目资料生成招商回复
- 能自动发送回复
- 能识别手机号、微信号、城市等留资信息
- 能创建和更新本地线索
- 能同步线索到飞书多维表格
- 能通知销售群
- 能在异常、登录失效、请求人工时停止自动回复并告警
